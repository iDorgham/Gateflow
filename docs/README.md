# GateFlow Docs v2 Overview

This `docs/` folder is the **v2 documentation set**, aligned with the canonical product spec in `PRD_v6.0.md`.

## What to read first

- **Product requirements:** `PRD_v6.0.md`  
  The single source of truth for GateFlow v6 — apps, features, security, resident portal/mobile, scanner rules, and marketing suite.

- **Plans & initiatives:** `docs/plan/`  
  - `plan/context/` — initiative ideas (e.g. `IDEA_docs_v2_refresh.md`).  
  - `plan/execution/` — per‑initiative plans (e.g. `PLAN_docs_v2_refresh.md`).  
  - `plan/learning/` — cross‑phase patterns, incidents, decisions (to be filled as we execute plans).

Upcoming, via the Docs v2 refresh plan:

- `docs/guides/ARCHITECTURE.md` — high‑level system architecture.
- `docs/guides/SECURITY_OVERVIEW.md` — security and multi‑tenancy.
- `docs/guides/DEVELOPMENT_GUIDE.md` — local setup and workflows.
- `docs/guides/ENVIRONMENT_VARIABLES.md` — env vars per app.

## Legacy docs

All legacy planning and spec documents have been archived under:

- `docs/archive/plan-legacy/**` — old `docs/plan/**` tree (v1/v5 era plans, specs, and guides).

These should be treated as **historical reference only**. New work should update:

- `PRD_v6.0.md` for product changes.
- `docs/guides/**` and `docs/plan/**` for documentation and planning.
