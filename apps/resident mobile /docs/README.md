# GateFlow Resident Mobile

The `resident-mobile` application is a native iOS and Android application built via React Native (Expo). It delivers the exact feature set of the web-based `resident-portal` but with the added benefits of deep OS integration, native push notifications, and biometric security.

## 🎯 Primary Objective
To offer a premium, frictionless experience for high-frequency residents. The native app allows for instant push notifications regarding gate arrivals and secure offline storage of personal entry credentials.

## ✨ Key Features
- **Push Notifications**: Immediate alerts (via Expo Push) when a visitor scans a pass generated by the resident, or when a delivery arrives.
- **Native Wallet Integration**: Support to add personal and visitor QR passes directly to Apple Wallet / Google Pay for near-field, offline scanning.
- **Offline Reliability**: Utilize local caching (SQLite) to ensure residents can always present their own static QR entry code even with zero cellular reception at the gate.
- **Biometric Locking**: Require FaceID / TouchID before launching the app to prevent unauthorized pass generation.

## 🛠 Tech Stack
- **Framework**: React Native + Expo (Managed Workflow)
- **Styling**: Nativewind / React Native `StyleSheet` (Tapping into `packages/ui/src/tokens.ts -> nativeTokens`)
- **Navigation**: React Navigation (or Expo Router)
- **Icons**: Lucide React Native
- **Storage**: `expo-secure-store` for JWTs, SQLite for deep caching.

## 📁 Folder Structure

```text
├── app/                  # Expo Router views

│   ├── _layout.tsx       # Root authenticated boundaries
├── components/           # Mobile specific UI (outside of shared packages/ui)
├── hooks/                # Push notification and auth hooks
├── assets/               # Local fonts, splash screens
├── eas.json              # EAS Build profiles
├── app.json              # Expo manifest (Bundle IDs, Permissions)

```

## 🚀 Getting Started

### Local Development
1. Download the **Expo Go** application on your physical device.
2. Run from the root workspace:
```bash
pnpm turbo dev --filter resident-mobile
```
3. Scan the generated QR code in your terminal with your device's camera (iOS) or inside the Expo Go app (Android).

### Deployment
Handled exclusively via Expo Application Services (EAS). Ensure the `eas.json` is configured correctly before pushing to App Store / Play Store. Refer to the `DEPLOYMENT_GUIDE.md`.
